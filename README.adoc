== What Is This?

K9iser wraps existing configuration files and deployment scripts into
**self-validating K9 contracts** — files that validate themselves before executing.

K9 (Self-Validating Components) enforces a must-just-nickel contract triad
with tiered safety levels (Kennel/Yard/Hunt). K9iser applies this to existing
infrastructure.

== How It Works

Point K9iser at your configs in `k9iser.toml`. K9iser:

1. Analyses existing configs (YAML, TOML, JSON, HCL)
2. Generates K9 contracts with embedded validation rules
3. Adds tiered safety: Kennel (safe), Yard (moderate), Hunt (powerful)
4. Provides cryptographic signing and verification
5. Self-validation runs before any deployment

== Key Value

* **Configs that validate themselves** — no external validator needed
* **Tiered safety levels** — sandbox dangerous operations automatically
* **Signed contracts** — cryptographic proof of authorised changes
* **Deploy with confidence** — invalid configs caught before execution
